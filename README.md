# 📈 stock — 종가배팅 데이터/스크리닝 프로젝트

거래대금·등락률(상승률) 데이터로 **주도섹터 · 주도테마 · 주도주**를 찾아 종가배팅에
활용하기 위한 파이썬 프로젝트입니다. 1단계는 **증권사 앱키/계좌 없이** 바로 쓸 수 있는
`pykrx` 기반 "장 마감 후 스크리너"로 시작합니다.

> ⚠️ 투자 판단·손실 책임은 본인에게 있습니다. 이 도구는 데이터 스크리닝 보조용입니다.

---

## 🔍 어떤 API가 가장 좋은가 (조사·검증 결론)

단일 API로 세 데이터 축(거래대금·등락률·조회수)을 모두 주는 곳은 없습니다. **3중 조합**이 정답입니다.

| 소스 | 적합도 | 핵심 |
|---|---|---|
| **pykrx + FinanceDataReader** | ⭐ 시작점 | 앱키·계좌 불필요. 특정일 **전종목 거래대금·등락률·일봉**을 한 번에 (✅검증) |
| **한국투자증권 KIS** | ⭐ 실시간 | 거래대금·등락률 **순위 무료 제공**(✅검증). 모의투자·공식 예제 풍부 |
| **키움 신규 REST** | 👍 좋음 | `theme_group_list` 등 **테마 축 강점**. 2025 출시라 성숙도 미검증 |
| **네이버페이 증권 크롤링** | 🔸 보조 | **조회수·테마의 사실상 유일 소스**. 비공식·ToS/DB권 리스크 |
| **토스증권 API** | ❌ 현재 부적합 | 순위·테마·조회수 엔드포인트 **전무**(✅검증). 정식 오픈 미정 |

**검증으로 확인된 핵심 사실**
- ✅ **조회수(인기검색)는 어떤 증권사 공식 API에도 없음** → 네이버 크롤링만 가능(순수 조회수 아님, "검색 인기도").
- ✅ **시장 "테마" 공식 표준 분류 없음** → 소스마다 구성종목이 달라 재현성 낮음.
- ✅ KIS 거래대금 순위는 전용 API가 아니라 `volume-rank` + `fid_blng_cls_code=3`(거래금액순).
- ⚠️ KIS 분봉은 **당일 전용**(과거 최대 ~1년) → 종가배팅 백테스팅은 **일봉으로 시작**이 현실적.
- ⚠️ 최신 `pykrx`(1.2.8)는 KRX 로그인 요구/차단으로 이 환경에서 빈 응답이 옵니다.
  그래서 **1단계 데이터는 안정적으로 동작하는 `FinanceDataReader`(전종목 스냅샷)로 구현**했습니다.
  pykrx는 주도섹터(업종지수)용 **선택 의존성**으로만 두었습니다.

### 데이터 소스 맵

| 필요한 데이터 | 1순위(현재 구현) | 키움 REST(계좌 보유, 확장) |
|---|---|---|
| 거래대금 순위 | **FDR `StockListing('KRX')`** 의 `Amount` | `ka10032`(거래대금상위) |
| 등락률(상승률) 순위 | **FDR `StockListing`** 의 `ChagesRatio` | `ka10027`(전일대비등락률상위) |
| 조회순위(관심도) | ✅ 키움 `ka00198` (`--views`) | **`ka00198`(실시간종목조회순위)** — HTS 0198과 동일. `bigd_rank` |
| 외인·기관 수급 | ✅ 네이버 `frgn`(`--flow`) / ✅ 키움 `ka10059`(`--kiwoom`) | `ka10059`(종목별투자자기관별) |
| 프로그램 순매수(종목) | ✅ 키움 `ka90013`(`--kiwoom`) | `ka90013` (※ 차익/비차익 분해는 종목단위 불가) |
| 프로그램 비차익(시장) | (미구현) | `ka90005`/`ka90010` (KOSPI/KOSDAQ **전체만** — 시장 필터용) |
| 주도테마 | (미구현) **신호 기반 분석으로 도출** — 아래 "테마 분석" 참고 | — |
| 백테스팅용 시세 | **FDR `DataReader`**(개별 종목 일봉) | 차트 TR |

> **⚠️ 정정**: 초기 결론과 달리 **조회순위는 키움 신규 REST `ka00198`으로 접근 가능**합니다(HTS 0198 = ka00198,
> 공식 공지·오픈소스 래퍼 2종 교차확인). 단 응답은 원시 조회수가 아니라 `bigd_rank`(빅데이터 순위)입니다.
> 또한 **종목단위 프로그램 "비차익" 순매수는 어떤 API로도 제공되지 않습니다**(차익/비차익은 시장 전체 집계만).

### 🧩 테마 분석 (신호 기반 도출 — API 고정 분류 아님)

매일 아래 신호를 종목에 부착·점수화한 뒤, 함께 움직이는 종목을 묶어 **"오늘의 주도테마/주도주"** 를 도출합니다.

1. **후보풀**: 거래대금·등락률·거래량 상위 + `ka00198` 조회순위(관심도)
2. **수급 부착**: 외국인·기관 순매수(`ka10059` 또는 네이버 `frgn` 크롤링)
3. **프로그램 부착**: 종목 프로그램 순매수(`ka90013`) — 비차익은 시장 전체 방향성 필터로만
4. **점수화**: 신호별 0~1 정규화 → 가중합, 여러 신호 동시 상위인 "교집합" 종목에 가점
5. **클러스터링/라벨링**: 업종·키워드·뉴스 태그로 겹치는 종목을 임시 테마로 묶음
6. **선정**: 테마 점수 상위 1~3 = 주도테마, 테마 내 최상위 = 주도주
7. **검증 루프**: 매일 저장 → 익일 수익률로 가중치 조정

## 🚀 빠른 시작

```bash
# 1) 가상환경(권장)
python -m venv .venv
.venv\Scripts\activate        # Windows PowerShell
# source .venv/bin/activate   # macOS/Linux

# 2) 의존성 설치
pip install -r requirements.txt

# 3) 실행 — 가장 가까운 영업일의 주도주 후보 스크리닝
python -m closebet
```

### 사용 예

```bash
python -m closebet                            # 최근 거래일 전체(KRX) 주도주 후보
python -m closebet --market KOSDAQ --min-change 5   # 코스닥, 등락률 5% 이상
python -m closebet --top 150 --min-value-eok 100    # 거래대금 상위 150, 100억 이상
python -m closebet --flow --flow-top 30       # 네이버 수급 결합 점수 랭킹(앱키 불필요)
python -m closebet --kiwoom --views           # 키움 공식 수급+프로그램+조회순위 결합(.env 필요)
python -m closebet --save                     # 결과를 data/leaders_YYYYMMDD.csv 로 저장
python -m closebet --sectors                  # 주도섹터 근사 표시(실험적, pykrx 필요)
python -m closebet -h                          # 전체 옵션
```

**스크리닝 로직**: 거래정지·무거래 제거 → (선택)우선주 제외 → 거래대금 상위 N개 압축
→ 최소 거래대금·등락률 조건 필터 → 등락률·거래대금 순 정렬 = **오늘의 주도주 후보**.

> 데이터는 FDR의 **최근 거래일 종가 스냅샷**입니다(임의 과거일 전종목 조회는 pykrx 정상화 또는
> 개별 종목 `DataReader` 반복 수집이 필요 — 백테스팅 단계에서 다룹니다).

### 🔌 키움 REST 연동 (3단계, 앱키 필요)

키움 `ka00198`(실시간종목조회순위=관심도) 신호를 붙입니다.

```bash
copy .env.example .env         # (PowerShell: Copy-Item .env.example .env)
# .env 에 KIWOOM_APPKEY / KIWOOM_SECRETKEY 입력

python -m closebet.kiwoom_probe          # 연결 스모크 테스트(토큰+ka00198)  ✅ 라이브 검증됨
python -m closebet --kiwoom              # 공식 수급(ka10059)+프로그램(ka90013) 결합 점수
python -m closebet --kiwoom --views      # 위 + 조회순위(ka00198) 관심도까지 결합
python -m closebet --kiwoom --themes     # 주도주 점수화 + 오늘의 주도테마 도출  ✅ 라이브 검증됨
```

**주의**: 이 도구는 **조회 전용**입니다. 키움 클라이언트는 주문 엔드포인트(`/ordr`·`/crdordr`) 호출을
코드 레벨에서 차단(`KiwoomOrderBlockedError`)하므로 실전 앱키로도 **매매가 나가지 않습니다.**

- 앱키/시크릿은 [openapi.kiwoom.com](https://openapi.kiwoom.com/) 로그인 후 발급.
- **`KIWOOM_ENV`는 앱키 종류와 일치해야 합니다**: 실전 앱키 → `real`, 모의 앱키 → `mock`.
  (키움은 실전/모의 앱키를 별도 발급. 불일치 시 토큰 발급 오류 `[8030: 투자구분...]`.)
- 조회순위·수급·프로그램은 **조회 전용**이라 `real`이어도 주문이 나가지 않아 안전합니다.
  주문 기능을 붙일 때만 모의 앱키(`mock`)로 먼저 검증하세요.
- `.env`는 커밋되지 않습니다. **앱키/시크릿을 코드·커밋·채팅에 절대 넣지 마세요.**

## ⚠️ 주의사항

- **실시간이 아닙니다.** FDR 스냅샷은 최근 거래일 종가 기준입니다.
  장중 실시간 신호가 필요하면 키움 REST API 등으로 확장하세요(로드맵 6단계).
- 과도한 반복 호출은 차단될 수 있어 대량 수집 시 `time.sleep`·캐싱을 넣으세요.
- 네이버 크롤링(조회수·테마)은 비공식이며 **ToS/데이터베이스권 리스크**가 있으니
  개인 학습·비공개 용도로만, 요청 간 딜레이를 두고 사용하세요.
- **증권사 앱키/시크릿은 절대 커밋하지 마세요.** `.gitignore`가 `.env`·`*.key` 등을 제외합니다.
